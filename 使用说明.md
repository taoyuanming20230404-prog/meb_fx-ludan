# 分销录单工具 — 使用说明

## 一、工具能做什么

根据 Excel 中的客户信息，自动打开浏览器，在 HHR 系统中完成：

- **流程一**：先输入手机号查是否**重单**；若为重单则只记录不重录；若非重单则填写表单并提交录单。
- **流程二**：不查重，在**分享页**直接逐条录单。

运行结束后会生成 **录单反馈表**（带「状态」列），流程一中若有重单会生成 **重单记录** 文本文件。

---

## 二、运行方式（二选一）

### 方式 A：Python 源码运行

1. 安装 **Python 3.10+**，并安装依赖：
   ```bash
   pip install -r requirements.txt
   ```
2. 安装 **Google Chrome**（与系统一致即可）。
3. 准备与 Chrome **主版本一致**的 **ChromeDriver**：
   - **推荐**：将 `chromedriver.exe` 放在 **`fx_ludan.py` 同目录**（或打包后放在 **exe 同目录**）；  
   - 或设置环境变量 **`FX_CHROMEDRIVER`**；或放入系统 PATH。
4. 在项目目录执行：
   ```bash
   python fx_ludan.py
   ```

### 方式 B：免安装 exe（推荐给无 Python 环境）

1. 在有 Python 的电脑上执行 **`build_windows.bat`** 生成 **`dist\fx_ludan.exe`**。
2. 将 **`fx_ludan.exe`** 复制到任意文件夹，双击运行。  
3. 同样需要本机安装 **Chrome**，并配置好 **ChromeDriver**（同上）。  

更详细的打包与分发说明见 **[打包说明.md](打包说明.md)**。

## 三、Excel 表格要求

| 列名 | 说明 |
|------|------|
| **号码** | 客户手机号（必填；为空则该行跳过） |
| **项目** | 项目描述，用于匹配下拉关键词及填写说明 |
| **城市** | 城市信息，用于地区选择 |
| **微信** | 选填 |

- 第一行为表头，列名需与上表一致。  
- 程序会按「项目」列做关键词匹配；若原文中含 `)`，会先取括号前一段再匹配（与录单逻辑一致）。

---

## 四、操作流程（步骤 1～5）

启动后按控制台提示操作：

1. **选择流程**（若源码中已固定为流程一，则不会出现此步；恢复交互需取消 `main()` 里对 `ask_user_select_flow()` 的注释）  
   - `1`：流程一（查重 + 录单）  
   - `2`：流程二（仅分享页录单）
2. **启动浏览器**：自动打开重单查询页；选流程二时会多开一个分享页标签。
3. **登录**：在浏览器中完成登录后，回到**黑色命令行窗口**按 **回车** 继续。
4. **选择 Excel**：弹出窗口，选择要处理的 `.xlsx` 文件。
5. **自动录单**：按行处理，控制台会显示 **进度 [当前/总数]** 及每行状态。

## 五、项目关键词如何匹配

匹配顺序（与 `std_keywords.txt`、`synonyms.json` 配合）：

1. **精确包含**：原文中出现某个标准词，取**最长**匹配。  
2. **同义词**：原文中包含 `synonyms.json` 里的 **key**，则映射为对应标准词（长 key 优先）。  
3. **模糊匹配**（需安装 `rapidfuzz`）：与标准词列表比对，达到阈值则取最佳项。  
4. **兜底**：以上都未命中时，固定为 **「玻尿酸」**。

- **`std_keywords.txt`**：每行一个标准项目名（与网页下拉一致为佳）。  
- **`synonyms.json`**：口语/简称 → 标准词；可放在 **exe/python 同目录** 覆盖内置配置（打包后 exe 同目录优先）。

离线检查匹配效果：

```bash
python test_project_match.py
# 或
python test_project_match.py "你的表格.xlsx"
```

会输出统计并在同目录生成 **`项目匹配测试结果.xlsx`**。

## 六、输出文件说明

| 文件/目录 | 说明 |
|-----------|------|
| **录单反馈+YYYYMMDD.xlsx** | 与所选 Excel **同目录**；在原表基础上增加列 **「状态」**：已录单 / 重单 / 跳过 / 失败。 |
| **重单记录_YYYY-MM-DD.txt** | 流程一当次运行中判为重单的客户摘要；**同一天多次运行**会自动 **_2、_3…** 后缀，避免覆盖。 |
| **YYYY-MM\重单记录_…** | 同上内容一份写在当月文件夹内（与上面使用相同计数后缀）。 |
| **YYYY-MM\** | 按自然月创建的目录，用于存放当月重单记录副本。 |

## 七、流程一：查重说明

- 输入手机号后程序会 **失焦** 该输入框，再等待数秒，由页面自动判断是否重单。  
- 若页面提示「该客户已被推荐」等，记为 **重单**，该行不录单，只写入重单记录。

## 八、常见问题

**1. 提示找不到浏览器 / 连接被拒绝**  
- 确认 Chrome 已安装且未崩溃；关闭浏览器后重新运行程序。  
- 确认 ChromeDriver 版本与 Chrome **主版本号**一致。

**2. 控制台出现 Chrome 内部报错（GCM、DEPRECATED 等）**  
- 多为浏览器自身日志，一般**不影响录单**，可忽略。

**3. 项目或城市下拉点不到**  
- 页面结构若变更，需改脚本里对应选择器（或联系维护人员）。  
- 选择项目/城市后程序已预留等待时间，网络慢时可适当加长。

**4. 打包后改词库**  
- 在 **fx_ludan.exe 同目录**放置 **`std_keywords.txt`**、**`synonyms.json`**，会覆盖程序内置文件。

**5. 未安装 tkinter**  
- 部分精简版 Python 无 tkinter，无法弹出选文件窗口；需使用完整 Python 安装或命令行指定路径（需自行改代码支持传参）。
